## Overview

Jasper Pet Shop uses TailwindCSS v4.2.1 with the Vite plugin for styling. The design system emphasizes luxury, elegance, and premium aesthetics for a pet furniture e-commerce platform.

## TailwindCSS Setup

TailwindCSS is configured through the Vite plugin:

```typescript vite.config.ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [react(), tailwindcss()],
})
```

<Note>
  TailwindCSS v4 uses the Vite plugin instead of PostCSS configuration, providing better performance and simpler setup.
</Note>

## Responsive Design

### Breakpoint System

TailwindCSS default breakpoints:

- `sm`: 640px
- `md`: 768px
- `lg`: 1024px
- `xl`: 1280px
- `2xl`: 1536px

## Performance Optimization

<Steps>
  <Step title="Purge unused styles">
    TailwindCSS automatically purges unused styles in production builds through the Vite plugin.
  </Step>
  
  <Step title="Use arbitrary values sparingly">
    Prefer standard Tailwind utilities over arbitrary values like `w-[237px]` to maximize CSS reuse.
  </Step>
  
  <Step title="Group hover states">
    Use `group` and `group-hover:` for parent-child hover interactions:
    
    ```typescript
    <div className="group">
      <img className="group-hover:scale-110" />
    </div>
    ```
  </Step>
</Steps>

## Best Practices

<CardGroup cols={2}>
  <Card title="Consistency" icon="check">
    Use the same spacing values throughout (4px grid: 4, 8, 12, 16, 24, 32, 48, 64)
  </Card>
  
  <Card title="Semantic Colors" icon="palette">
    Use color classes semantically (primary, secondary, accent) rather than specific hex values
  </Card>
  
  <Card title="Mobile First" icon="mobile">
    Write base styles for mobile, then add breakpoint modifiers for larger screens
  </Card>
  
  <Card title="Reusable Components" icon="cube">
    Extract repeated patterns into React components rather than duplicating classes
  </Card>
</CardGroup>
